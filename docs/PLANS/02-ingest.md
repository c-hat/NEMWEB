## Finalisation target

A clean **source-adapter** layer: each external source (AEMO/NEMWEB, OpenElectricity,
weather, market/dispatch, static reference) is an adapter producing **normalized
dataset payloads** with stable semantics, independent of any visualisation shape.
Adapters write to R2 (raw + normalized) and record runs in D1 via the storage
layer (workstream 3); during migration they keep emitting the current
`public/data` compatibility files.

## Dependencies

- **Storage & Catalog (03)** for where normalized/raw payloads land (R2) and how
  runs/availability are recorded (D1). Until that exists, keep writing files.
- Coordinates with **Analysis (05)** on the normalized dataset contract that
  analyses consume.

## Tasks (ordered)

### Phase A — close out the current ingest

1. **Resolve the 17:00-vs-16:00 decision** ([`FLAGS.md`](../../FLAGS.md) §3).
   This is a product call (ask the owner). Once decided, fix `_forecast_cutoff`
   and document the rationale in `ingest/README.md`; add a test asserting the
   chosen snapshot is picked.
2. **Backfill robustness.** `Reports/Current/` rolls off; add an `ARCHIVE/`
   fallback (currently a documented limitation) or make skips explicit and
   visible. Add a test for the rolled-off-day path.
3. **DST / timezone note.** AEST fixed at +10:00 is a documented prototype limit.
   Either keep and assert it, or plan localisation — record the decision; do not
   leave it ambiguous in the normalized contract.

### Phase B — define the adapter/dataset boundary

4. **Extract a `SourceAdapter` interface.** Generalise the existing `Source` so
   NEMWEB is one adapter among several. An adapter: fetches raw, parses, and
   emits a **normalized dataset** (not a per-day frontend file). Keep NEMWEB
   behaviour identical.
5. **Define the normalized dataset contract** in
   [`docs/DATA_CONTRACTS.md`](../DATA_CONTRACTS.md): a dataset has id, metric,
   cadence, region set, interval grid, units, and per-interval values with
   `null` for missing — decoupled from the frontend per-day shape.
6. **Make the frontend per-day file a derived view** of normalized datasets, not
   the primary output. The existing `public/data/*.json` becomes a compatibility
   projection. Tests assert the projection equals today's output byte-for-byte
   for the validated day.

### Phase C — wire to Cloudflare storage (with workstream 3)

7. **Write raw + normalized payloads to R2** behind the storage module; record a
   **source run** (id, source, params, started/finished, status, object refs) in
   D1. Keep emitting compatibility files until the Worker API serves from R2.
8. **Promote `fetch_live.py` to a live adapter** that writes the live payload to
   storage (read by `GET /api/live`) instead of force-pushing the `live-data`
   branch. Phase the branch out only after the API path is proven.

## Contracts to honour

- 48 half-hour-ending intervals, AEST `+10:00`, `null` for missing.
- POE mapping confirmed in [`FLAGS.md`](../../FLAGS.md): `poe10 ← POWERPOEHIGH`,
  `poe90 ← POWERPOELOW`; `poe10 >= poe50 >= poe90`.
- Demand actual = unadjusted `OPERATIONAL_DEMAND`; rooftop actual = `POWER` with
  `TYPE=MEASUREMENT` only.
- Adapters are source-specific; the normalized dataset contract is not.

## Acceptance criteria

- `uv run python -m pytest -q` passes; new tests cover backfill rolloff and the
  chosen snapshot cutoff.
- The compatibility per-day projection is byte-identical to current output for
  2026-05-28.
- A second adapter (even a stub, e.g. OE demand or weather) can be added without
  touching NEMWEB code, proving the seam.
- Live + historical payloads can be written to R2 and recorded in D1 (Phase C),
  with the file outputs still produced during migration.

## Guardrails

- Do not change React view logic.
- Do not encode visualisation-specific shapes as an adapter's only output.
- No broad edits to generated `public/data`; treat it as derived output.
- Keep the same code path for live site and fixtures (the `Source`/adapter seam).
